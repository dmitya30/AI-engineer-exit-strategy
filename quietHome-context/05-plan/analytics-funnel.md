## Пять шагов воронки

```
[1] Сайт → [2] Запуск бота → [3] Код Души бесплатно → [4] Экран оплаты → [5] Платёж завершён
```

### Шаг 1. Заход на сайт numerologpro.ru

**Что меряем:** уникальные визиты, источник трафика, поведенческие (время на сайте, глубина просмотра, scroll до блока тарифов).

**Где смотрим:** Я.Метрика, отчёт «Источники сводка» + Вебвизор для качественной оценки.

**Источники трафика, которые ожидаем:**
- Прямые заходы (после анонсов в каналах).
- Telegram (переходы из @quantocode, @panevin_channel, канала Наташи, канала Саныча).
- Поиск (низкочастотные запросы - результат через 4-8 недель, в этом спринте маловероятен).
- Дзен (если будут дубли статей с canonical).

**Минимальная цель в Я.Метрике, которую настроить:**
- «Заход на сайт» (стандартный визит) - автоматически.
- «Scroll до блока тарифов» - JS-цель (см. ниже).
- «Клик по CTA в бот» - JS-цель (см. ниже).
- «Заход на /about/» - URL-цель.
- «Заход на /blog/» - URL-цель.

### Шаг 2. Переход с сайта в бот (запуск бота)

**Что меряем:** клики по CTA-кнопкам с UTM-метками + событие `/start` в боте.

**Где смотрим:**
- Я.Метрика - клики (event-цель «Клик по CTA в бот»).
- Бот-логи - событие первого запуска для нового user_id.

**UTM-метки на сайте** (из `context-v2.1-ch1.md` §5):
- Hero: `?utm_source=site&utm_medium=hero&utm_campaign=main`
- Pricing: `?utm_medium=pricing&utm_campaign={free|basic|check|generate|expert}`
- Stories: `?utm_medium=stories&utm_campaign=main`
- Corners: `?utm_medium=corners&utm_campaign=quantocode`
- Footer: `?utm_medium=footer&utm_campaign=main`
- /about/: `?utm_medium=about&utm_campaign=main`

**Важно:** UTM в URL Telegram не передаются как query - отслеживаем клики через Я.Метрику. В боте источник определяется через параметр `start` (если настроен).

**Расхождение:** клики в Метрике ≠ запуски в боте. Часть кликов теряется (открыли ссылку, не нажали Start). Это нормально - смотрим обе цифры отдельно.

### Шаг 3. Расчёт Кода Души (бесплатный шаг)

**Что меряем:** сколько пользователей дошли до `SOUL_RESULT` - получили свой Код Души.

**Где смотрим:** бот-логи на n8n, фильтр по состоянию `SOUL_RESULT`.

**Воронка внутри бота на этом шаге:**
```
/start → AWAITING_BIRTHDATE → ввод даты → SOUL_RESULT
```

**Где теряются пользователи:**
- Запустили бот, не ввели дату.
- Ввели дату некорректно, не повторили попытку.
- Получили результат, ушли.

**Целевая метрика:** конверсия запуск → результат. Здоровый показатель - 60-80% (бесплатный шаг, низкий барьер).

---

### Шаг 4. Показ экрана оплаты 490 ₽

**Что меряем:** сколько пользователей увидели экран с предложением полного расчёта трёх кодов (490 ₽ / 325 ⭐).

**Где смотрим:** бот-логи, состояние `MAIN_MENU` → переход в `buy_basic_*`.

**Это первая платная точка.** До этого момента пользователь ничего не платил.

**Где теряются:**
- Увидели цену 490 ₽, не нажали оплату.
- Нажали оплату, не дошли до окна ЮKassa.

**Целевая метрика:** конверсия результат → показ экрана оплаты. Здоровый показатель в эзо-нише - 10-25% (большинство останавливается на бесплатном).

---

### Шаг 5. Завершённый платёж

**Что меряем:** платежи в статусе `succeeded` в ЮKassa + событие `PAYMENT_SUCCESS_*` в боте.

**Где смотрим:**
- ЮKassa - кабинет, отчёт по платежам.
- Бот-логи - событие успешной оплаты.
- «Мой налог» - сверка по выданным чекам (ручная выдача, см. `context-v2.1-ch1.md` §4).

**Важно:** считаем только сторонних лиц. Платежи от родственников, друзей семьи и команды (Дима/Наташа/Саныч) - не засчитываем в KPI спринта.

**Целевая метрика:** конверсия показ экрана → оплата. Здоровый показатель - 15-30% от тех, кому показали экран.

## Целевые цифры спринта

**Главный KPI спринта:** ≥ 1 платящий стороннее лицо к 28.06.2026.

**Ориентир-максимум** (из §10 ch2, скорректирован под старт с 29.05):

| Метрика | День 14 (12.06) | День 30 (28.06) |
|---|---|---|
| Подписчики @quantocode | +5 | +30 |
| Бесплатные расчёты Кода Души | 5 | 20 |
| Показы экрана оплаты 490 ₽ | 1-2 | 5-10 |
| Платящие basic/check | 0 | 1 |
| Платящие Эксперт | 0 | 0 |

**Если цифры выше ориентира** - хорошо, готовимся к масштабированию.
**Если ниже ориентира, но ≥ 1 платящий к дню 30** - спринт выполнен по главному KPI, остальное - данные для итерации.
**Если 0 платящих к дню 30** - анализ по матрице в `plan-30-days-current.md`, решение о следующей итерации.

## Что считается «здоровьем» воронки

| Шаг | Метрика | Здоровый коридор |
|---|---|---|
| Сайт → клик в бот | Конверсия от визита | 5-15% |
| Клик в бот → /start | Доходимость | 50-70% (часть теряется на переходе) |
| /start → Код Души посчитан | Конверсия бесплатного шага | 60-80% |
| Код Души → экран оплаты | Конверсия в платный | 10-25% |
| Экран оплаты → платёж | Конверсия оплаты | 15-30% |

**Если на каком-то шаге конверсия резко ниже коридора** - это место сломано, чинить именно его, не выше и не ниже.

## Цели в Я.Метрике (настроены 01.06.2026)

В Я.Метрике настроены 4 цели:

1. **Заход на /about/** - URL-цель, condition: URL содержит `/about/`. Идентификатор: `visit_about`.
2. **Заход на /blog/** - URL-цель, condition: URL содержит `/blog/`. Идентификатор: `visit_blog`.
3. **Клик по CTA в бот** - JavaScript-цель. Идентификатор: `cta_bot`. Скрипт ловит клики по всем ссылкам с `t.me/quantocode_bot` через делегирование на document.
4. **Scroll до блока тарифов** - JavaScript-цель. Идентификатор: `scroll_pricing`. Скрипт следит за появлением `#pricing` во viewport, срабатывает один раз за визит.

Скрипты живут в Code Injection → Site Footer. Параллельно события дублируются в GA4 (события `cta_bot_click`, `scroll_pricing`).

«Заход на сайт» как отдельная цель не настраивается — Метрика считает визиты автоматически в основном отчёте.

Сводка по целям доступна в отчёте «Конверсии» Я.Метрики.

## Сверка с ЮKassa

Раз в неделю (суббота, в рамках формального среза):
1. Открыть кабинет ЮKassa, выгрузить список платежей за неделю.
2. Сверить с бот-логами (event `PAYMENT_SUCCESS_*`).
3. Проверить, что все чеки выданы через «Мой налог» (триггер 422-ФЗ - в день расчёта).
4. Записать итоги в `state-current.md` (короткая строка: дата, сумма, статус чека).

## Что НЕ меряем

- Лайки, реакции, охваты постов сами по себе - это не воронка, это шум.
- Подписчики в @panevin_channel и канале Саныча - не наша метрика спринта.
- Время на сайте сверх 2 минут - не значимый показатель для лендинга.
- Bounce rate главной - на лендинге это нормально, главное - дошли ли до CTA.
