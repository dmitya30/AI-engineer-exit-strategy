# Контекстный документ: ТИХИЙ ДОМ. Оркестрация

**Версия:** 1.0
**Дата:** 2026-04-29
**Тип диалога:** стратегический (оркестрация)
**Назначение:** стратегическая рамка проекта "Тихий Дом", приоритеты, контроль исполнения. Тактика и разработка - в связанных диалогах.

## 1. Контекст и происхождение

Совместный проект Дмитрия Паневина и Александра Романова (Саныча). Параллельный трек к основному направлению Дмитрия (услуги Б3/Б4). Цель - полупассивный доход на готовых, но не запущенных эзотерических продуктах.

Стартовая база: ноль выручки (ELYOR ~3000₽ за 9 месяцев), три технически готовых продукта, отсутствие бренда, аудитории и бюджета. Бюджет ~10К₽ из стихийных поступлений на неопределенный период.

## 2. Рабочее название

"Тихий Дом" - зонтичный бренд экосистемы инструментов для эзотерических практиков и их клиентов. Тон: спокойный, профессиональный, без крикливости, в пику типичному эзо-рынку. Статус: временно-финальное, пересмотр при упаковке.

## 3. Стратегическая рамка

**Долгосрочное видение (12+ месяцев):** портал-агрегатор для эзо-практиков с подпиской, общей базой клиентов и витриной продуктов. Растет органически из локомотива, не строится с нуля.

**Среднесрочный фокус (3-6 месяцев):** один продукт-локомотив + два сателлита, монетизирующиеся внутри воронки локомотива.

**Краткосрочный фокус (60 дней):** запуск B2B-версии локомотива и проверка гипотезы платящих профи.

## 4. Продуктовая архитектура

**Локомотив - КВАНТОКОД (B2B):**
SaaS для нумерологов. Подписочная модель, оплата через ЮKassa. Активный маркетинг направлен сюда. Защитный ров: проприетарная методика квантовой нумерологии + рабочие артефакты для профи (PDF, база клиентов, шаблоны).

**КВАНТОКОД (B2C):**
Бесплатный расчет Кода Души + платные расчеты в Telegram Stars для физиков. Точка входа в экосистему через органический трафик и через клиентов нумерологов.

**Сателлит 1 - Mystic Oracle:**
Подключается внутри воронки КВАНТОКОДа как "углубление" для клиентов нумерологов. Отдельный маркетинг не ведем. Уязвим к LLM-копированию - усиливать защитный ров постепенно.

**Сателлит 2 - ELYOR:**
Подключается как "персональный амулет на основе кодов" после расчета. Партнерская модель: нумеролог получает комиссию за заказ от своего клиента. Отдельный маркетинг не ведем.

## 5. Каналы и точка сборки

**Основная точка сборки:** собственный сайт "Тихий Дом" (отдельный домен/поддомен, не panevin.net). Архитектура - landing + личный кабинет, тариф, оплата, контентный блок.

**Каналы доставки:**
- ТГ-канал КВАНТОКОДа (рабочий) - контент, новости, кейсы
- Дублирование в Max и ВК - резерв на случай ужесточения блокировок
- Посевы в нумерологических чатах (компетенция Саныча)
- SEO как долгосрочная игра (сильная сторона Дмитрия)
- Разовый "запуск через жену" в ее сообществах (один раз, без давления)

## 6. Защитный ров и упаковка

Две части одной системы.

**Защитный ров (что нельзя скопировать):**
- проприетарная методика расчета (LLM не знает)
- структурированные артефакты (PDF, шаблоны)
- интеграция в рабочий процесс профи (база, история, отправка клиенту)
- партнерские/сетевые механики

**Упаковка (как клиент захочет купить):**
- язык боли нумеролога: "выглядеть профессиональнее", "успевать больше клиентов", "красивые отчеты для клиентов"
- личная история: "технарь сделал инструмент для жены-нумеролога, перерос в SaaS"
- демонстрация результата: до/после, скриншоты PDF, видео-разбор
- доверие: кейс жены как первого пользователя, отзывы первых нумерологов

## 7. Монетизация (рабочие гипотезы, тестируем)

**B2B-нумерологи:**
- триал 7 дней за 0-300₽
- месячная подписка 1500-3000₽
- годовая 12000-25000₽ (скидка 30-40%)
- оплата через ЮKassa

**B2C-физики:**
- сохраняем текущие тарифы в Telegram Stars (325/532/1529)
- пересматриваем после теста основной гипотезы

**ELYOR:**
- разовая покупка персонального амулета, цена обоснованная (не донат)
- партнерская комиссия нумерологу при заказе от его клиента

**Mystic Oracle:**
- подписка через звезды или ЮKassa, пересборка freemium-границы (бесплатного должно быть мало)

## 8. Юрлицо и платежи

Старт - на самозанятости Дмитрия. Подключение ИП Саныча, самозанятости жены - откладываем до момента, когда денежный поток оправдает усложнение.

## 9. Распределение ролей

**Дмитрий:** стратегия, разработка, продукт, контент-производство (с AI), SEO, оркестрация.

**Саныч:** маркетинг, посевы, продажи, общение с первыми клиентами B2B, опционально - инфлюенс-маркетинг.

**Жена:** первый живой пользователь для кейса, контент-донор (1-2 раза в месяц получасовое интервью), валидация UX и текстов с точки зрения профи, разовый запуск в своих сообществах. Не продает, не ведет клиентов.

**AI-агенты:** контент, копирайтинг, аналитика, рутина.

## 10. Метрики первой итерации (60 дней)

- **Целевая (амбициозная):** 30-35 платящих нумерологов, MRR ~100К₽.
- **Реалистичный диапазон:** 5-15 платящих, MRR 15-45К₽.
- **Минимум для подтверждения гипотезы:** 3-5 платящих + понимание, что есть масштабирование.
- **Провал:** 0 платящих за 60 дней - пересобираем гипотезу.

## 11. Принципы работы

- Один продукт в активном маркетинге, остальные - "бонусом" в воронке
- Не множить гипотезы, проверять одну за раз
- Стратегия не строится на ресурсах, которые не контролируем (в т.ч. на активности жены)
- Защитный ров от LLM-копирования - обязательное требование к продукту
- Сайт - основная точка сборки, ТГ - канал доставки

## 12. Что НЕ делаем

- Не строим зонтичный портал с нуля
- Не льем отдельный трафик на Mystic Oracle и ELYOR
- Не запускаем все три продукта параллельно с маркетингом
- Не давим на жену с продажами
- Не используем платную рекламу эзотерики (Директ/ВК) на старте
- Не делаем продукты, легко повторимые в чате с LLM

## 14. Связь с другими диалогами

**Типы диалогов** (по системе "Правил v3.0", Д1-Д4 = типы, не конкретные диалоги):
- Д1: позиционирование, офферы, язык боли
- Д2: клиенты, продажи, воронка
- Д3: контент и площадки
- Д4: техническая разработка

**Диалоги основного направления Дмитрия** не пересекаются с "Тихим Домом" - это параллельный трек.

**Диалоги проекта "Тихий Дом":**
- **ТИХИЙ ДОМ. Оркестрация** (исходный диалог) - стратегия, приоритеты, контроль, декомпозиция
- **ТИХИЙ ДОМ. Разработка** (создать) - технический диалог под разработку сайта, инфраструктуры
- **КВАНТОКОД. Разработка** (текущий) - технический диалог под рефакторинг КВАНТОКОДа
- **ТИХИЙ ДОМ. Контент** (создать при необходимости) - контент-стратегия, тексты, посты, SEO
