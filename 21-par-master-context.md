## 1. Клиент

**Имя:** Елена Сивова
**Кто:** Пар-мастер (банные практики), самозанятая
**Связь с Дмитрием:** старый личный контакт, общение на "ты"
**Сегмент:** Б4 (услуги) с уклоном в Б3 (личный бренд эксперта-практика)
**География:** Новосибирск, Бердск, Алтай (разъездная модель)

**Текущая ситуация:**
Работает по найму в академических банях. Личного потока заявок нет - клиенты "просто приходят" по факту присутствия. Поток `2-3 человека в рабочий день`, рабочие дни не каждый день. Целевой поток - `2 заявки в рабочий день` (потолок по физике, работает одна). Приоритет - индивидуальные сеансы (дороже, объёмнее).

**Ключевой драйвер проекта:**
Переезд на Алтай работать пар-мастером летом 2026. Бани съёмные, 3 локации. Модель: разные дни - разные города (Новосибирск / Бердск / Алтай). Клиенту нужно видеть, в какой день где Елена, чтобы состыковать со своим отпуском. На низком старте - может уехать в любой момент.

---

## 2. Задача

Единая точка приёма заявок - сайт. Клиент с QR (визитка, реклама на машине) заходит, видит позиционирование + расписание по городам, оставляет заявку. Заявка падает Дмитрию... (Елене) в Telegram + в таблицу. Дальше - ручная переписка в удобном клиенту канале.

**Что осознанно НЕ делаем:**
❌ Ботов в мессенджерах (Telegram/WhatsApp/Max/ВК) - дорого, хрупко, не закрывает расписание по городам
❌ WhatsApp-автоматизацию - нелегально на личном аккаунте + RU-блокировки звонков с 08.2025, риск полной блокировки
❌ AI-агента в виджете на старте - риск галлюцинаций про услуги/цены, избыточно при потоке 2-3/день
❌ CRM на MVP - таблица + Telegram закрывают всё
❌ Жёсткое бронирование слотов - нужен только "календарь доступности" (ориентировочная запись, финал стыкуется вручную)
❌ Привязку к конструктору Т-Банка - делаем свой контур на VPS

---

## 3. Решение и этапы

📌 **Этап 1. Сайт + приём заявок** - `30 000`, срок 1-1,5 нед.
Лендинг (1 стр.), форма-квалификатор как пошаговый диалог (мгновенные ответы, без LLM), сбор данных (был/не был, индивид/компания, даты, предпочтительный канал связи + контакт), заявка в Telegram + таблицу, домен/хостинг/QR.

📌 **Этап 2. Расписание по городам + панель** - `35 000`, срок 1-2 нед.
Self-service панель для Елены (правит расписание сама), отображение "город + доступные дни" клиенту, запись с привязкой к городу/дате, Telegram-напоминание об истечении расписания. Запускается под выезд на Алтай ("скажет еду - стартую").

✅ **Юридика 152-ФЗ** - бесплатно (вынесена из цены)
Шаблоны (политика, согласие, оферта) бесплатно, подачу в РКН клиент делает сам на pd.rkn.gov.ru. Дмитрий не юрист - позиционируется как "рабочий минимум", не аудит.

⏩ **На потом (след. год, отдельно):** Яндекс.Метрика + сбор аудитории по РФ под Алтай. Реклама на машине у клиента уже есть (QR делаем в Этапе 1).

## 5. Техническая архитектура (для Д4)

**Стек:** свой бэкенд на VPS (Python/FastAPI или Node), статичный фронт с виджетом-формой, хранилище заявок (SQLite или Google Sheets), уведомления через Telegram Bot API. n8n НЕ используется (избыточен для задачи).
**Внешние зависимости:** только Telegram Bot API (легально, стабильно).
**Критичное требование:** дублирование заявок (таблица/БД + Telegram-пуш) - не полагаться только на пуш, иначе пролистанная заявка = потерянный клиент.
**Виджет:** пошаговый сценарий на кнопках/полях, оформленный как живой диалог. БЕЗ LLM на MVP. AI-слой - возможный этап 3 (+`20-30К`), только при росте потока.
**Расписание (Этап 2):** self-service панель (выбран клиентом из 3 вариантов - предпочёл полный визуальный контроль без посредников-таблиц), модель "календарь доступности" read-only для клиента, не бронирование слотов.

## 6. Правовой контур (152-ФЗ)

Клиент собирает ПД (контакты) -> становится оператором ПД. Актуально на 2026:
📌 Штраф за обработку без согласия (физлица/самозанятые): `400 000-500 000`
📌 Штраф за неподачу уведомления в РКН: до `300 000`
📌 Ужесточение действует с 30.05.2025
Решение: шаблоны документов на сайт + самостоятельная подача уведомления через pd.rkn.gov.ru. Ответственность за юридику Дмитрий на себя не берёт (не юрист) - явно проговорено в КП.
